Image File Format Comparison

PNG PNG (Portal Network Graphic) is a lossless file format, developed in 1997 as a replacement for GIF. It supports RGB, RGBA and Grayscale colour modes.

99.7MB

 PNG is an ideal format for Quality, since it preserves the original images exactly and has excellent compatibility on modern systems. However it has the largest file size on this list so it may not be good in cases where file size is important.
TIFF TIFF (Tag Image File Format) is a image container format, supporting lossy and lossless images, developed in 1986 by Adobe (then Aldus Corporation). It also supports many other advanced features such as layers and sub-files, and has many "extensions" which add more features, meaning compatibility is poor and files can be tricky to read.

99.5MB

 TIFF in this case does have excellent quality, and has a slightly lower file size than PNG. However, the difference isn't noticeable, and quality is identical, and since TIF doesn't work even in a lot of modern browsers, never mind older applications, means it's impossible to recommend.
JPG 50% Quality JPG is a very popular lossy file format, particularly for photography. Since it has a variable level of compression, it can be used for both high quality use cases and situations where getting the smallest filesize possible is most important. It was introduced in 1992 and has been the most widely used compression standard since.

1.6MB

 JPEG with 50% quality looks fairly decent and would hold up just fine as long as quality is not being scrutinised. It also has a much lower file size that PNG or TIF, making it perfect for most websites where quality is not important.
25% Quality

1.2MB

 At 25% quality, the image is noticably degraded and looks quite poor. This would make it unsuitable for anything but small images such as thumbnails, where the poor quality would be harder to notice. However it would be useful displayed any larger than this.
5% Quality

0.35MB

 At 5% quality, the image looks awful. The colours are significantly reduced and the image barely resembles the original. The level of compression would be useless on anything but a very old system where small size if of paramount important, and while the filesize is tiny it's reaching the point of dimishining returns on modern systems as most modern internet connections could load even the 50% quality JPEG in a matter of seconds.
